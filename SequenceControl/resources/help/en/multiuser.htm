Collaborative Design

A collaborative server is used to provides the function of collaborative design. The collaborative server is in charge of the communication among all applets (applications) connected to the server. The information of communication includes user operation messages, chat messages, and management of token. Online chat is used to enhance the communication. Due to the security limitation, a applet can only be connected to the web server machine. Therefore, the collaborative server and web server are in the same computer.

The MultiUser menu perform collaboratie design function. It has following submenu:

  • Connect: Connect to the collaborative server.
  • Disconnect: Disconnect from the collaborative server.
  • Grab token: Grab the token to dominate the design process.
  • Release token: Release the token so that other user can get it.
  • Synchronize: Load the circuit design from the collaborative server to update to the latest status.
  • chat: Open a chat windows to talk to others.

First of all, user connects to the collaborative server. Once connected, the program load the current circuit design from the collaborative server. After that, any modification of the circuit design made by the user or others will be shown in the program.

A token is used to restrict the design process. Only the user who get the token can modify the circuit design.

Online chat is used to talk to all users who are connected to the collaborative server.
